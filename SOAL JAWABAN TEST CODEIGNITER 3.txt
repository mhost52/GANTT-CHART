## 📌 Bagian A – Teori CodeIgniter 3
1. **MVC**  
   - *Model*: mengelola data dan logika bisnis.  
   - *View*: menampilkan data ke pengguna.  
   - *Controller*: penghubung antara model dan view, menerima input dari user.  
   - Data berpindah: User → Controller → Model → Controller → View → User.

2. **Routing**  
   - Definisi di `application/config/routes.php`, contoh:  
     ```php
     $route['users/(:num)'] = 'UserController/detail/$1';
     ```  
   - Keuntungan: URL lebih rapi, fleksibel, dan aman.

3. **Security**  
   - CSRF: aktifkan di `config.php` → `$config['csrf_protection'] = TRUE;`.  
   - XSS Filtering: gunakan `$this->input->post('field', TRUE);`.

4. **Session Management**  
   - Load library: `$this->load->library('session');`.  
   - File-based: default, disimpan di server.  
   - Database-based: lebih aman, cocok untuk aplikasi skala besar.

5. **Helper vs Library**  
   - Helper: kumpulan fungsi procedural (misalnya `form_open()`).  
   - Library: class OOP dengan state dan method (misalnya `email`).  
   - Contoh: `form_input('username');`.

## 📌 Bagian B – PostgreSQL
1. **Normalisasi**  
   - 1NF: data atomic.  
   - 2NF: setiap kolom bergantung pada primary key.  
   - 3NF: tidak ada *transitive dependency*.  
   - Foreign key:  
     ```sql
     ALTER TABLE orders ADD CONSTRAINT fk_user FOREIGN KEY(user_id) REFERENCES users(id);
     ```

2. **Query Optimization**  
   - `EXPLAIN SELECT * FROM users;` → menampilkan rencana eksekusi query.  
   - Index: `CREATE INDEX idx_users_status ON users(status);`.

3. **Data Types**  
   - `CHAR(n)`: fixed length.  
   - `VARCHAR(n)`: variable length dengan batas.  
   - `TEXT`: panjang tidak terbatas.  
   - `JSONB`: binary storage, lebih cepat untuk query dibanding `JSON`.

4. **Transactions**  
   - ACID: Atomicity, Consistency, Isolation, Durability.  
   - `COMMIT`: menyimpan perubahan.  
   - `ROLLBACK`: membatalkan transaksi.

5. **Function**  
   ```sql
   CREATE OR REPLACE FUNCTION calc_discount(price NUMERIC, percent NUMERIC)
   RETURNS NUMERIC AS $$
   BEGIN
     RETURN price - (price * percent / 100);
   END;
   $$ LANGUAGE plpgsql;
   ```

## 📌 Bagian C – Praktik
1. **Konfigurasi DB** (`application/config/database.php`)  
   ```php
   $db['default'] = array(
     'dsn'   => '',
     'hostname' => 'localhost',
     'username' => 'postgres',
     'password' => 'yourpassword',
     'database' => 'yourdb',
     'dbdriver' => 'postgre',
     'dbprefix' => '',
     'pconnect' => FALSE,
     'db_debug' => TRUE,
   );
   ```  
   - Query builder:  
     ```php
     $this->db->where('status', 'active');
     $query = $this->db->get('users');
     ```

2. **CRUD**  
   - Model:  
     ```php
     public function insert_user($data) {
       return $this->db->insert('users', $data);
     }
     ```  
   - Controller:  
     ```php
     public function delete($id) {
       $this->User_model->delete_user($id);
     }
     ```

3. **Error Handling**  
   - Undefined index: gunakan `isset()` atau `array_key_exists()`.  
   - Error query: aktifkan `$db['default']['db_debug'] = TRUE;` atau tangkap dengan `try-catch`.
