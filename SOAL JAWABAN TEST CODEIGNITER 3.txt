## 📌 Bagian A – Teori CodeIgniter 3
1. **MVC**  
   - *Model*: mengelola data dan logika bisnis.  
   - *View*: menampilkan data ke pengguna.  
   - *Controller*: penghubung antara model dan view, menerima input dari user.  
   - Data berpindah: User → Controller → Model → Controller → View → User.

2. **Routing**  
   - Definisi di `application/config/routes.php`, contoh:  
     ```php
     $route['users/(:num)'] = 'UserController/detail/$1';
     ```  
   - Keuntungan: URL lebih rapi, fleksibel, dan aman.

3. **Security**  
   - CSRF: aktifkan di `config.php` → `$config['csrf_protection'] = TRUE;`.  
   - XSS Filtering: gunakan `$this->input->post('field', TRUE);`.

4. **Session Management**  
   - Load library: `$this->load->library('session');`.  
   - File-based: default, disimpan di server.  
   - Database-based: lebih aman, cocok untuk aplikasi skala besar.

5. **Helper vs Library**  
   - Helper: kumpulan fungsi procedural (misalnya `form_open()`).  
   - Library: class OOP dengan state dan method (misalnya `email`).  
   - Contoh: `form_input('username');`.

## 📌 Bagian B – PostgreSQL
1. **Normalisasi**  
   - 1NF: data atomic.  
   - 2NF: setiap kolom bergantung pada primary key.  
   - 3NF: tidak ada *transitive dependency*.  
   - Foreign key:  
     ```sql
     ALTER TABLE orders ADD CONSTRAINT fk_user FOREIGN KEY(user_id) REFERENCES users(id);
     ```

2. **Query Optimization**  
   - `EXPLAIN SELECT * FROM users;` → menampilkan rencana eksekusi query.  
   - Index: `CREATE INDEX idx_users_status ON users(status);`.

3. **Data Types**  
   - `CHAR(n)`: fixed length.  
   - `VARCHAR(n)`: variable length dengan batas.  
   - `TEXT`: panjang tidak terbatas.  
   - `JSONB`: binary storage, lebih cepat untuk query dibanding `JSON`.

4. **Transactions**  
   - ACID: Atomicity, Consistency, Isolation, Durability.  
   - `COMMIT`: menyimpan perubahan.  
   - `ROLLBACK`: membatalkan transaksi.

5. **Function**  
   ```sql
   CREATE OR REPLACE FUNCTION calc_discount(price NUMERIC, percent NUMERIC)
   RETURNS NUMERIC AS $$
   BEGIN
     RETURN price - (price * percent / 100);
   END;
   $$ LANGUAGE plpgsql;
   ```

1. **Konfigurasi DB** (`application/config/database.php`)  
   ```php
   $db['default'] = array(
     'dsn'   => '',
     'hostname' => 'localhost',
     'username' => 'postgres',
     'password' => 'yourpassword',
     'database' => 'yourdb',
     'dbdriver' => 'postgre',
     'dbprefix' => '',
     'pconnect' => FALSE,
     'db_debug' => TRUE,
   );
   ```  
   - Query builder:  
     ```php
     $this->db->where('status', 'active');
     $query = $this->db->get('users');
     ```

2. **CRUD**  
   - Model:  
     ```php
     public function insert_user($data) {
       return $this->db->insert('users', $data);
     }
     ```  
   - Controller:  
     ```php
     public function delete($id) {
       $this->User_model->delete_user($id);
     }
     ```

3. **Error Handling**  
   - Undefined index: gunakan `isset()` atau `array_key_exists()`.  
   - Error query: aktifkan `$db['default']['db_debug'] = TRUE;` atau tangkap dengan `try-catch`.

```sql
JOIN orders o ON u.id = o.user_id
```

```sql
CREATE MATERIALIZED VIEW monthly_sales AS
SELECT DATE_TRUNC('month', sale_date) AS month,
       SUM(amount) AS total_sales
FROM sales
GROUP BY DATE_TRUNC('month', sale_date);

REFRESH MATERIALIZED VIEW monthly_sales;
```

# 📊 Kunci Jawaban Singkat

**1. JOIN User & Orders**
```php
public function get_orders_with_users() {
  $this->db->select('orders.id as order_id, users.name as user_name, orders.order_date, orders.total_amount');
  $this->db->from('orders');
  $this->db->join('users', 'users.id = orders.user_id');
  return $this->db->get()->result();
}
```

**2. JOIN Products & Categories**
```php
$this->db->select('products.name as product_name, categories.category_name, products.price');
$this->db->from('products');
$this->db->join('categories', 'categories.id = products.category_id');
$this->db->where('categories.category_name', 'Electronics');
$this->db->where('products.price >', 1000000);
$query = $this->db->get()->result();
```

**3. JOIN + Aggregasi**
```sql
SELECT o.id AS order_id, u.name AS user_name,
       SUM(oi.quantity * oi.price) AS total_amount
FROM orders o
JOIN users u ON u.id = o.user_id
JOIN order_items oi ON oi.order_id = o.id
GROUP BY o.id, u.name;
```

**4. Pagination**
```php
public function list($page = 1) {
  $limit = 10;
  $offset = ($page - 1) * $limit;
  $this->db->select('orders.id, users.name, orders.order_date, orders.total_amount');
  $this->db->from('orders');
  $this->db->join('users', 'users.id = orders.user_id');
  $this->db->limit($limit, $offset);
  $data['orders'] = $this->db->get()->result();
  $this->load->view('orders/list', $data);
}
```

**5. LEFT JOIN + COALESCE**
```php
$this->db->select('orders.id, COALESCE(users.name, "Unknown") as user_name, orders.order_date, orders.total_amount');
$this->db->from('orders');
$this->db->join('users', 'users.id = orders.user_id', 'left');
$query = $this->db->get()->result();
```
