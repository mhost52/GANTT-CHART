Maksud dari **“user/group management terbatas pada file sharing, quota, dan basic permission”** di NAS adalah bahwa pengaturan hak akses biasanya hanya berfokus pada hal-hal mendasar terkait file dan folder. Contohnya:

### 1. **File Sharing**
- Admin bisa membuat folder bersama (misalnya: `QC_Data`, `Produksi`, `Manajemen`).
- Setiap folder bisa diatur siapa saja yang boleh **akses**:
  - User A: hanya bisa **read-only** (melihat file, tidak bisa edit).
  - User B: bisa **read & write** (edit, tambah file).
  - User C: **full control** (hapus, ubah permission).

### 2. **Quota**
- NAS biasanya punya fitur **membatasi kapasitas per user**.
  - Misalnya: User QC hanya diberi quota 50 GB.
  - User Produksi diberi quota 200 GB.
  - Tujuannya supaya tidak ada satu user yang menghabiskan seluruh storage.

### 3. **Basic Permission**
- Level izin biasanya sederhana:
  - **Read** → hanya bisa membuka file.
  - **Write** → bisa menambah/mengedit file.
  - **Execute** → jarang dipakai di NAS, tapi ada untuk script/aplikasi.
- Tidak sekompleks server yang bisa punya kombinasi ACL (Access Control List) detail, inheritance, dan integrasi dengan sistem keamanan enterprise.

- Folder `QC_Report`:  
  - User QC → Read-only (supaya laporan tidak diubah sembarangan).  
  - User Produksi → Read & Write (bisa update data produksi).  
  - User Manajemen → Full control (bisa hapus, ubah struktur folder).

- User QC boleh baca file `.pdf` tapi tidak boleh buka file `.xlsx`.  
- User Produksi boleh edit file di jam kerja saja (pakai policy).

Betul Billy, maksudnya NAS umumnya **tidak bisa kontrol sampai ke level file yang sangat detail** seperti server.  

📌 **Penjelasan:**
- **NAS**:  
  - Permission biasanya diatur per **folder share**.  
  - Misalnya: folder `QC_Report` → User QC hanya bisa **read-only**, User Produksi bisa **read & write**.  
  - Kalau ada 100 file di dalam folder itu, semua file akan ikut aturan yang sama.  
  - Beberapa NAS modern memang mendukung ACL (Access Control List), tapi biasanya terbatas dan tidak sekompleks server.  

- **Server (Windows/Linux)**:  
  - Bisa atur permission **per file**.  
  - Contoh: dalam folder `QC_Report`, file `laporan_maret.pdf` bisa di-set read-only untuk QC, tapi file `laporan_april.xlsx` bisa di-set editable untuk Produksi.  
  - Bisa juga bikin aturan inheritance (aturan folder turun ke file, tapi bisa di-override).  
  - Bahkan bisa kombinasikan dengan policy lain: misalnya file tertentu hanya bisa diakses dari IP kantor, atau hanya di jam kerja.  

👉 Jadi, NAS lebih cocok untuk **kontrol sederhana di level folder**, sedangkan server memberi fleksibilitas penuh sampai ke level file, waktu akses, dan integrasi dengan sistem keamanan enterprise.
